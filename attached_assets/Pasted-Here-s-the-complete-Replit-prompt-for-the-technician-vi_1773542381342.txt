> **CMMS Technician Task View — Complete Mobile Screen**
>
> Implement a dedicated mobile task detail screen for technicians. This is a completely separate view from the admin task detail — technicians have no access to Edit, Delete, subtask management, or any admin controls. Route to this screen when the logged-in user has the `technician` role. The screen is full-height, single-column, and scrollable with a sticky bottom bar.

> - 3px progress bar below label row (`#4338CA` fill, transitions)

> - Checked state: `#4338CA` filled square with white checkmark, name gets strikethrough + `#9CA3AF` color
> - **Locked state** (before Start Task): entire list at opacity 0.4, pointer-events none
> - Lock banner at bottom of card: lock icon + "Start the task to unlock subtasks" in `#9CA3AF`, `#F8F8F8` bg

> ### Card 3 — Checklist

> ### Card 4 — Notes + Photos (single card)
> - **Notes section:**
>   - Section label "NOTES" on left. Auto-save indicator on right — shows "Saving…" in `#4338CA` while debounce is pending, then "Saved" in `#15803D` for 2 seconds, then disappears. Both are 10px.
>   - Open `<textarea>`: no border, transparent bg, 13px, `#1A1A1A`, placeholder `#C4C9D4`. Placeholder text: "Type your observations… auto-saved as you type". Min 3 rows, auto-expands.
>   - **Auto-save behavior**: debounce 1200ms after last keystroke. On save: capture current textarea value + timestamp, clear textarea, insert saved note as a `#F8F8F8` rounded card (8px border-radius, padding 8px 10px) below the textarea showing the note text (12px, `#1A1A1A`) and timestamp (10px, `#9CA3AF`). Multiple notes stack newest-first. No save button — saving is fully automatic.
>
> - **Photos section** (within the same card, below notes):
>   - Photos only appear after the first one is taken — no placeholder, no empty state shown
>   - Each photo: 36×36px rounded square (border-radius 6px), pastel fill cycling through `#BFDBFE` → `#BBF7D0` → `#FED7AA` → `#FECDD3` → `#DDD6FE`, white camera SVG icon inside (14×14px)
>   - Photos wrap in a flex-wrap row with 5px gap
>   - Photos are added by tapping the camera button in the bottom bar (see below)

> ## STATE MACHINE
>
> ### NOT STARTED
> - Hero: Not Started gradient + "Not Started" badge
> - Timer: hidden
> - Subtasks: locked, dimmed, lock banner visible
> - Camera button: disabled, gray, non-interactive
> - CTA: "▶ Start Task"
>
> ### IN PROGRESS (after Start Task)
> - Hero: In Progress gradient, badge updates to "In Progress"
> - Timer: appears top-right of hero, starts counting up from 0:00
> - Subtasks: unlocked, full opacity, interactive, lock banner hidden
> - Camera: enabled, `#4338CA` color
> - CTA: "⏸ Pause"
>
> ### PAUSED (after Pause)
> - Hero: Paused gradient, badge updates to "Paused"
> - Timer: stops counting, keeps current value
> - CTA: "▶ Resume"
> - Camera: stays enabled
>
> ### RESUMED (after Resume)
> - Returns to IN PROGRESS state, timer continues from where it paused
>
> ### COMPLETED (after Mark as complete)
> - Timer stops
> - Full-screen overlay replaces entire view:
>   - `#F0FDF4` background
>   - 62×62px green circle (`#15803D`) with white checkmark
>   - "Task Complete" 18px weight 700 `#15803D`
>   - Task name 13px `#166534`
>   - "Time logged: Xm Xs" in `#4ADE80`
>   - "Back to Tasks" button (`#15803D` bg) — navigates to task list
>
> ---
>
> ## BEHAVIOR RULES
>
> - Notes auto-save with 1200ms debounce — no save button. Each saved note appends to history below textarea with timestamp. Textarea clears after save.
> - Timer starts on Start Task, pauses on Pause, resumes on Resume. Final elapsed time submitted on completion as "time logged."
> - Photos are only addable after Start Task. Each camera tap adds one 36×36px thumbnail inline in the notes card.
> - Subtasks locked until Start Task. Technician cannot add or rename subtasks.
> - Checklist always interactive regardless of task state.
> - Parent task back-link only shown if this task has a parent. Taps navigate up to the parent task detail screen.
> - This screen is only rendered for `technician` role. Admin users always see the admin task detail view. Never mix the two views.
> - All existing routing, back navigation, and session management must remain intact.
